# Markdown creation and editing

## 概要

このスキルは Markdown ファイルの作成と編集における一貫した記述ルールを提供します。ドキュメントの可読性と保守性を向上させ、プロジェクト全体で統一された形式を保ちます。

Markdown ドキュメントは技術仕様書、設計書、README など様々な用途で使用され、明確な構造と統一された記述規則により、チーム全体での情報共有を効率化します。

## ワークフロー判断

### 新規ドキュメント作成

* テンプレートの選択
	* 設計書: `docs/design.md.sample` をベースに作成
	* 一般ドキュメント: 以下の記述規則に従って作成
* 見出し構造の設計
	* H1 でドキュメントタイトルを定義
	* H2-H3 で論理的な階層構造を構築
	* H4 以降は使用せず、リストのネストで表現
* 内容の組み立て
	* MECE 原則で情報を整理
	* 関連する内容をセクションでグループ化

### 既存ドキュメント編集

* 既存の記述規則に準拠
	* 見出しレベルの維持
	* リスト形式の統一（アスタリスク使用）
	* スペーシングルールの遵守
* 内容の整理
	* MECE 原則での再構成
	* 論理的な順序での配置
* コードブロックの保持
	* 既存のコードは作業証跡として保持
	* 改変せずそのまま維持

## 記述規則

### 見出し

* H1 はドキュメントのタイトルとして利用します
* H2 から H3 までを使用します
* H4 以降は基本的に使用しません
* 見出しに連番（1. や 1.1 など）を使用しません
* 見出しを細かく切らず、リストのネストで階層構造を表現します

実装例:

修正前:
```markdown
# システム設計
#### 詳細なコンポーネント設計
##### フロントエンド詳細
##### バックエンド詳細
```

修正後:
```markdown
# システム設計

## コンポーネント設計

* フロントエンド
	* UI コンポーネント
	* 状態管理
* バックエンド
	* API エンドポイント
	* データベース設計
```

### リスト

* アスタリスク（*）を使用します
* ハイフン（-）は使用しません
* ネストはタブを使用します
* 手順を示す場合は番号付きリストを使用できます

実装例:

修正前:
```markdown
- 項目1
  - サブ項目1-1
    - サブサブ項目1-1-1
```

修正後:
```markdown
* 項目1
	* サブ項目1-1
		* サブサブ項目1-1-1
```

### 強調表示

* 強調表示（太字・斜体）は使用しません
* 構造と階層で情報の重要度を表現します

実装例:

修正前:
```markdown
**重要**: この設定は必須です
*注意*: バックアップを取得してください
```

修正後:
```markdown
## 重要事項

* この設定は必須です

## 注意事項

* バックアップを取得してください
```

### コードブロック

* コードブロックには適切な言語指定を行います
* インラインコードは ` で囲みます

実装例:

修正前:
```markdown
以下のコマンドを実行:
```
npm install
```
```

修正後:
````markdown
以下のコマンドを実行:
```bash
npm install
```
````

インラインコード例:
```markdown
`package.json` ファイルを編集してください
```

### テキスト記述

* スペーシング
	* 英語（アルファベット）と日本語の間には半角スペースを一つ入れます
	* これにより可読性が向上します
	* 例:
		* 修正前: `LINEを使って`
		* 修正後: `LINE を使って`
		* 修正前: `GitHubリポジトリ`
		* 修正後: `GitHub リポジトリ`

## 内容規則

### コードブロックの扱い

* コードブロック内のコードは作業結果を貼り付けたものです
* 既存のコードブロックの内容は改変しません
* 作業の証跡として保持します

例:

```markdown
## 実装結果

以下のコードを実装しました:

```javascript
function example() {
  return "This code block should not be modified";
}
```

→ このコードブロックは編集せず、そのまま保持
```

### 内容の整理

* 内容は MECE（漏れなく重複なく）にまとめます
* 関連する情報は同じセクションにグループ化します
* 論理的な順序で記述します

整理の原則:

1. 重複の排除
	* 同じ情報を複数箇所に記載しない
	* 関連情報は一箇所にまとめる
2. 漏れの防止
	* 必要な情報をすべて含める
	* チェックリストで確認
3. 論理的な順序
	* 時系列順、重要度順など明確な基準で配置
	* 読み手の理解しやすい流れを意識

## コードスタイルガイドライン

Markdown ドキュメント作成時は以下を遵守してください:

* 簡潔な記述
	* 冗長な説明を避ける
	* 必要な情報のみを含める
* 一貫性の保持
	* プロジェクト全体で同じルールを適用
	* 既存ドキュメントのスタイルに準拠
* 可読性の優先
	* 階層構造を明確にする
	* 適切な空行で視認性を向上
